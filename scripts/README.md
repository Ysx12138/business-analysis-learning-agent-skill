# Scripts Directory

## Recommended Entry Point

**`run_analysis.py`** — Universal analysis runner.

This is the only script you should use for new analyses. It handles:

- single CSV/Excel file input
- multi-table folder input (auto-detects relationships, builds join plan, merges)
- field semantics inference (100+ patterns)
- metric auto-detection (14 base + derived metrics)
- analysis planning (ranking, trend, distribution)
- Excel + designed HTML + PDF output generation

```bash
python3 scripts/run_analysis.py --input /path/to/data.csv --mode beginner_summary
```

## Core Modules (`core/`)

Reusable analysis modules consumed by `run_analysis.py`:

| Module | Responsibility |
|--------|---------------|
| `data_intake.py` | Load CSV/Excel, basic profiling (row count, missing values, types) |
| `field_semantics.py` | Infer field business meaning from naming patterns (100+ patterns, 16 domains) |
| `metric_registry.py` | 14 registered base metrics + derived metrics (CTR, CPC, profit margin, etc.) |
| `analysis_planner.py` | Generate analysis plan: grouping, ranking, trend, distribution, cross-analysis |
| `relationship_detector.py` | Multi-table: single/compound key detection, join plan, safe merge execution |
| `result_schema.py` | Unified dict schema (v0.2.0) consumed by all renderers |

## Renderers (`renderers/`)

Reusable output renderers:

| Renderer | Format | Library |
|----------|--------|---------|
| `excel_renderer.py` | `.xlsx` | openpyxl |
| `html_report_renderer.py` | `.html` / `.pdf` | HTML/CSS + WeasyPrint / Playwright |
| `pdf_renderer.py` | `.pdf` fallback | reportlab |

All active renderers consume the same result schema from `result_schema.py`.

The recommended PDF path is `html_report_renderer.py`: it writes a browser-reviewable
`*_report.html` file first, then exports PDF with WeasyPrint or Playwright. `pdf_renderer.py`
is kept only as a stable fallback when HTML-to-PDF export is unavailable.

Presentation generation is not part of the universal runner. Every presentation must be
generated by an Agent after reading the selected bundled presentation Skill under
`vendor/agent_presentation_skills/`.

## Legacy Scripts (`legacy/`)

Historical validation and generation scripts from earlier iterations (v0.1–v0.5). These are kept for reference in `scripts/legacy/` and are **not recommended** for new analyses.

See `scripts/legacy/README.md` for the full list.

Use `run_analysis.py` instead for any new analysis.
