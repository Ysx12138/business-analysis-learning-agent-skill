# Customization Guide

## 1. Customize for Different User Levels

- beginner: more explanations and simpler methods
- intermediate: balanced explanation and method depth
- advanced: concise explanation with optional deeper analysis

## 2. Customize for Different Industries

Adapt examples, metrics, and assumptions for:

- retail
- marketing
- operations
- finance
- education

## 3. Customize Output Templates

Modify files in `templates/` to match your reporting style and required structure.

## 4. Add New Test Cases

Create a new folder under `test_cases/` with:

- `input_description.md`
- `expected_behavior.md`
- `sample_dataset_schema.md`
- `sample_output.md`

## 5. Extend the Metric Glossary

Add industry-specific metrics to `templates/metric_glossary_template.md` with meaning, formula, use, and common misunderstanding.

## 6. Localize the Skill

You can provide Chinese, bilingual, or other language versions while preserving the same workflow structure.
